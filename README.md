Этот проект запускается внутри контейнера Docker на базе "python:3.11-slim".
Цель — быстро поднять среду для запуска скрипта "run.py".

Как поднять за 3 минуты
- Построить образ
docker build -t stats-app .

- Запустить контейнер
docker run --rm stats-app --start-date 2025-01-01 --end-date 2025-01-31

Так как в run.py используется argparse с обязательными параметрами --start-date и --end-date,
их нужно передавать прямо в команду запуска контейнера.
Пример запуска через Docker:
docker run --rm stats-app --start-date 2025-01-01 --end-date 2025-01-31


Если хочешь включить режим планировщика (--schedule), добавь этот флаг:
docker run --rm stats-app --start-date 2025-01-01 --end-date 2025-01-31 --schedule

Контейнер автоматически выполнит python run.py благодаря ENTRYPOINT.

- docker run --rm stats-app — запускает контейнер и удаляет его после завершения.
- ENTRYPOINT ["python", "run.py"] в Dockerfile гарантирует, что внутри контейнера всегда выполняется python run.py.
- Всё, что ты добавляешь после имени образа (--start-date ... --end-date ...) автоматически передаётся как аргументы в run.py.
Таким образом, проект можно запускать с любыми датами и опцией планировщика прямо через Docker.

Структура проекта
- run.py — главный скрипт для запуска
- main.py — основная логика
- StatsDB.py — модуль работы с базой статистики
- DataLoader.py — загрузка данных
- StatUpdater.py — обновление статистики
- fb_spend.json — данные расходов Facebook
- network_conv.json — данные конверсий сети
  
Зависимости
Устанавливаются напрямую при построении образа:
- apscheduler
- pytest
  
Тестирование
Запустить интерактивный контейнер и выполнить тесты:
docker run -it --rm stats-app pytest
